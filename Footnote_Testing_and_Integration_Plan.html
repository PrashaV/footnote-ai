Testing Claim-to-Citation Matching

1

Confirm the backend server is running

In your terminal, activate the venv and start the FastAPI server. You should see Uvicorn running on port 8000.

# Terminal 1 — backend source ~/footnote-venv/bin/activate cd "/Users/prashavelmurugan/Documents/Claude/Projects/Footnote: AI powered Research Intelligence/backend" uvicorn main:app --reload --port 8000
2

Prepare a test draft with real, verifiable citations

The claim matcher only works when citations resolve to real papers with abstracts in Semantic Scholar. Use this known-good test draft — it contains a real paper that Semantic Scholar indexes.

# Example test draft (copy this into the app's text area) Attention mechanisms have become a dominant approach in sequence modeling. Vaswani et al. (2017) showed that transformer models relying entirely on self-attention outperform recurrent architectures on machine translation tasks, achieving 28.4 BLEU on WMT 2014 English-to-German translation. The model eliminates recurrence entirely, allowing greater parallelization during training compared to RNN-based approaches.
Real citation: Vaswani et al., 2017 Indexed in Semantic Scholar Has a public abstract
3

Submit and check the Citations tab first

Before looking at Claim Accuracy, confirm the citation resolved. In the Citations tab, Vaswani et al. 2017 should show status "verified" and have a non-null abstract field. If it shows "unverified", the claim matcher cannot run — the abstract is missing.

If citation shows "unverified": Check that SEMANTIC_SCHOLAR_API_KEY is set in backend/.env. Unauthenticated requests are heavily rate-limited. A key lifts this significantly — get one free at semanticscholar.org/product/api.
4

Check the Claim Accuracy tab

Click the "Claim Accuracy ✦" tab. You should see a verdict for the Vaswani et al. sentence. Expected result: Supported (the abstract does confirm transformers outperform RNNs on translation). If it shows "Unverifiable", the abstract was not returned by Semantic Scholar — retry or use a different paper.

5

Test an overstated claim

Now test a deliberately overstated version of the same claim to confirm the matcher catches misrepresentation.

# Overstated version — change "outperform" to "completely replace" Vaswani et al. (2017) proved that transformer models have completely replaced all recurrent architectures in every NLP task, making LSTMs and GRUs obsolete.

Expected verdict: Overstated — the paper showed better translation BLEU scores but did not claim to replace all recurrent architectures in every NLP task.

6

Test a contradicted claim

Test a claim that directly contradicts what the paper says.

# Contradicted version Vaswani et al. (2017) demonstrated that recurrent neural networks are superior to attention-based models for machine translation tasks.

Expected verdict: Contradicted — the paper's conclusion is the exact opposite of this claim.

7

Test via the raw API (optional — for debugging)

If the UI is not showing results, call the API directly to see the raw JSON and isolate whether the issue is backend or frontend.

curl -X POST http://localhost:8000/api/verify \ -H "Content-Type: application/json" \ -d '{ "draft": "Vaswani et al. (2017) showed transformers outperform RNNs on translation.", "title": "Test", "check_citations": true, "check_claim_matching": true, "check_ai_writing": false }' | python3 -m json.tool | grep -A 20 "claim_match"
What to look for: claim_match.total_checked should be > 0 and claim_match.verdicts should contain at least one entry. If total_checked is 0, the citation did not resolve with an abstract.

Testing AI Writing Detection (Sapling)

1

Get your Sapling API key

Go to sapling.ai/user/login → sign up → Dashboard → API Keys → create a persistent key (not the 24-hour trial key shown on first login). Copy the key.

Free tier: No credit card required. Generous enough for testing. The key does not expire as long as you use the "create key" flow rather than the demo key.
2

Add the key to your .env file

echo 'SAPLING_API_KEY=your_key_here' >> \ "/Users/prashavelmurugan/Documents/Claude/Projects/Footnote: AI powered Research Intelligence/backend/.env"

Then restart the backend server (Ctrl+C, then run uvicorn again) so it picks up the new env var.

3

Test with known human-written text

Use a passage you wrote yourself — something personal, conversational, with varied sentence length. Expected result: score < 20%, verdict "Likely Human".

# Good test: personal writing, conversational, varied rhythm When I first started studying machine learning, I kept getting confused by the terminology. Every paper seemed to invent its own vocabulary. I remember reading about "latent representations" for weeks before realizing it just meant compressed features. The field has a habit of making simple ideas sound more complex than they are — sometimes on purpose, sometimes not.
4

Test with known AI-generated text

Ask ChatGPT or Claude to write a paragraph on any academic topic, then paste it in. Expected result: score > 65%, verdict "Likely AI" on clearly AI-generated text.

Important caveat: AI detectors have a false positive rate — some human writing scores high, especially formal academic prose. A high score is a flag to review, not proof. The disclaimer in your UI already covers this.
5

Test graceful degradation (no key)

Temporarily remove the key from your .env and submit a draft. The AI Writing tab should show "AI detection unavailable — SAPLING_API_KEY not configured" rather than crashing. Restore the key afterward.

6

Verify flagged passages appear correctly

Submit a mixed draft (half human, half AI). The AI Writing tab should show individual sentences scored high as flagged passages, with severity badges (high / medium / low). Confirm the sentence text matches your draft.

7

Test via raw API

curl -X POST http://localhost:8000/api/verify \ -H "Content-Type: application/json" \ -d '{ "draft": "YOUR DRAFT TEXT HERE", "check_citations": false, "check_claim_matching": false, "check_ai_writing": true }' | python3 -m json.tool | grep -A 30 "ai_writing"

Check that ai_writing.score is a number 0–100, verdict is one of likely_human / uncertain / likely_ai, and disclaimer mentions Sapling.

Adding Real Plagiarism Detection (Turnitin / iThenticate)

1

Understand what's available (and what's not)

Real plagiarism detection requires matching submitted text against a massive document database. Only a few vendors have this. The main options for API integration are:

Provider API Available Best For Cost
Turnitin ✓ iThenticate API v2 Academic institutions, publishers Enterprise pricing — contact sales
Copyleaks ✓ REST API Startups, developers, affordable Pay-per-scan or monthly plans (~$10/mo)
PlagiarismCheck.org ✓ REST API SMBs, affordable Credit-based
Writer.com ✗ No public API Web use only —
Recommendation for Footnote: Start with Copyleaks — it has a proper REST API, developer-friendly docs, and affordable pricing that makes sense for an early-stage product. Turnitin requires institutional contracts.
2

Sign up for Copyleaks API access

Go to copyleaks.com/sign-up → create an account → go to API Access in your dashboard. You'll get an email and an API key. The free tier gives you a small number of scans per month to test with.

API docs: api.copyleaks.com Free trial scans available
3

Understand the Copyleaks API flow

Copyleaks works asynchronously — you submit text, get a scan ID, then poll or receive a webhook when the scan completes. The flow is:

  • Step A: POST to /v3/education/submit/file/{scanId} with your text as a base64-encoded file
  • Step B: Poll /v3/education/{scanId}/status until status == 410 (completed)
  • Step C: GET /v3/education/{scanId}/result for the full report with matched sources and similarity %
This is not instant. Copyleaks scans take 30 seconds to several minutes. You'll need to handle this asynchronously — either with polling on the backend, or by returning a "scan pending" state to the frontend and letting the user refresh.
4

Add Copyleaks credentials to .env

# Add to backend/.env COPYLEAKS_EMAIL=your-copyleaks-account-email COPYLEAKS_API_KEY=your-copyleaks-api-key

Also add these to backend/.env.example (without values) so they're documented.

5

Create backend/services/plagiarism_service.py (rewrite)

Replace the current stub with a real Copyleaks integration. The service needs three things:

  • authenticate() — POST to /v3/account/login/api with email + key, returns a bearer token (valid 48 hrs, cache it)
  • submit_scan(draft, scan_id) — submit the text for scanning
  • get_result(scan_id) — poll until complete, return similarity score + matched sources

Tell Claude: "Write a plagiarism_service.py that integrates the Copyleaks API — authenticate, submit text scan, poll for result, return a PlagiarismResult with similarity_score (0–100), matched_sources list, and status."

6

Update the data models

Add a PlagiarismResult model to backend/models/verify.py and add plagiarism: PlagiarismResult | None to IntegrityReport. Mirror the TypeScript types in frontend/src/api/verifyTypes.ts.

# New Pydantic model to add to models/verify.py class PlagiarismMatchedSource(BaseModel): url: Optional[str] title: Optional[str] similarity: float # 0.0–1.0 class PlagiarismResult(BaseModel): similarity_score: float # 0–100, overall matched_sources: list[PlagiarismMatchedSource] scan_id: str status: Literal["completed", "pending", "failed"] disclaimer: str
7

Wire it into verify_service.py

Add Copyleaks as a 4th parallel task in run_verification(). Because it's async, run it alongside AI detection and unsupported claims. Update _compute_scores() to include a plagiarism dimension in the overall score (suggested weight: 10–15%, reducing other weights slightly).

Handle timeouts gracefully. Copyleaks can take 2–3 minutes. Set a generous timeout (120s) but let the scan fail gracefully — the rest of the report should still return without it. Show "Plagiarism scan timed out — try again" in the UI rather than blocking the whole report.
8

Build a PlagiarismPanel frontend component

Create frontend/src/components/integrity/PlagiarismPanel.tsx (replacing the current stub). It should show:

  • A similarity score gauge (lower = better for the user)
  • A list of matched sources with URLs and similarity percentages
  • A disclaimer that this is powered by Copyleaks and what it checks

Add a "Plagiarism" tab to IntegrityDashboard.tsx that renders this panel when report.plagiarism is non-null.

9

Add an honest disclaimer in the UI

Even with Copyleaks, be transparent about what it checks. Add a note like:

"Plagiarism check powered by Copyleaks — matches against web pages, published papers, and student submissions in their database. Results reflect similarity, not intent. A high similarity score may reflect legitimate quotation or common phrases. Always review matched sources in context."
10

Test end-to-end with a known-plagiarized draft

Submit a draft containing a few sentences copied verbatim from a public Wikipedia article (e.g., the opening paragraph of "Attention (machine learning)" on Wikipedia). Expected result: Copyleaks should flag that passage with a similarity match pointing to the Wikipedia URL.

Test type: known positive Expected: similarity > 30% Expected source: en.wikipedia.org
